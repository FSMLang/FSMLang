C Language Output Generators

C Language Ouput Generators are based on two sub-classes of the FSM Output Generator. One sub-class is used for the top-level machine; the other, for sub-machines. The sub-machine class is a sub-class of the top-level machine class.

The top-level machine adds a (pointer to) a data structure and function pointers for use to facilitate the common code generation pattern which must be tailored for each C design output and, further, for the different action function return types. These pointers are populated appropriately when the classes are instantiated by the factory, or when the base class writeMachine function is called.

Common Language Elements Generated

Language elements are sensative to three different dimensions: The "level" of the machine, which includes the idea of whether sub-levels exist; the return type of the action functions; and, the requested C Design type. The later is indicated on the command line and forms the basis for determining the first choices in the C Language Output Generator's sub-class.

When an item is indicated through the use of the simple 'X' as being present in different levels of machine, it may be that the element has parts that are only present in a given level. If a machine comprises two levels, it will have the parts required by each level.

Destination Files

In keeping with good modularization, interface information is published on an as needed basis. Code wanting to use the FSM needs only to know the run function and the event definition. This information is published in the <base_file_name>.h file, where <base_file_name> is either the output file name or the machine name. Because experience has shown that the event enumeration (not the event structure, in the case of events with data) is of use separate from any other machine information, that enumeration is published in <base_file_name>_events.h. This is, of course, included by the public header.

Interface information needed by files holding user functions is published in <machine_name>_private.h, where <machine_name> is taken from the relevant machine.
